# 内网穿透说明

> 本地开发完成后，可用穿透把本机 API / 前端暴露给同组成员或答辩环境。

## 1. 先本地跑通

1. MySQL：`building_manos` + `scripts/setup-db.ps1`
2. API：`scripts/run-api.ps1` → `http://127.0.0.1:8080/api/health`
3. 前端：`scripts/run-web.ps1` → `http://127.0.0.1:5173`

## 2. 推荐穿透方案

### A. 只穿透 API（推荐）

前端同学本机跑 Vite，`VITE_API_BASE_URL` 指向你的 API 隧道地址。

```powershell
# 示例：ngrok
ngrok http 8080
```

前端 `frontend/.env.development`：

```env
VITE_API_BASE_URL=https://xxxx.ngrok-free.app
```

重启 `pnpm dev`。后端已配置 CORS `*` 与 `host=0.0.0.0`。

### B. 同时穿透前端与 API

| 本地端口 | 用途 |
|----------|------|
| 8080 | Java API |
| 5173 | Vite 前端 |

分别开两条隧道。前端构建/开发时设置：

```env
VITE_API_BASE_URL=https://<api-tunnel-host>
```

注意：浏览器页面域名与 API 域名不同时必须依赖 CORS（已默认放行）。

### C. frp 示例

```ini
[building-api]
type = tcp
local_ip = 127.0.0.1
local_port = 8080
remote_port = 18080
```

客户端访问 `http://<frp-server>:18080/api/health`。

## 3. 检查清单

- [ ] `GET /api/health` 返回 `"db":"UP"`
- [ ] 前端系统状态页显示 API CONNECTED
- [ ] 能完成一次购买并在成交记录中看到票据
- [ ] 勿把 `database.properties` 里的真实密码提交到公开仓库

## 4. 安全提示

穿透仅用于开发联调与答辩演示，勿在未鉴权环境长期暴露公网。
