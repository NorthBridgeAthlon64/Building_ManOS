## 2. 分层架构对照（鲜花商店 → 本系统）

课堂**鲜花商店**是大作业最接近的参考实现，对比如下：

| 鲜花商店 | Building ManOS | 说明 |
|----------|----------------|------|
| `entity/` | `model/` | 实体类，对应数据库表 |
| `dao/` + `dao/impl/` | `dao/` | 数据访问；本项目简化为具体类，不强制接口+impl 拆分 |
| `BaseDao` | `config/DBConfig` + dao 内工具 | 连接获取、资源关闭；可抽 `DaoSupport` 基类（可选） |
| `service/` + `service/impl/` | `service/` | 业务逻辑 |
| `test/Main.java` | `cli/MenuController` + `Main` | 控制台菜单；**本项目要求 cli 不直接 new Dao** |
| `database.properties` | `src/main/resources/database.properties` | 数据库连接配置 |
| `Buyable` / `Sellable` 接口 | `DiscountStrategy` | 接口 + 多实现，策略模式 |
| `account` 成交账目表 | `sale_record` | 购买记录落库 |

### 本项目的改进点（相对课堂鲜花商店）

课堂 `Main.java` 中部分逻辑**直接 new Dao**，本项目中：

- ✅ `MenuController`（cli）只调用 `XxxService`
- ✅ 业务判断（能否删除、能否购买）放在 `service`
- ✅ 统一使用 `PreparedStatement`，不用字符串拼接 SQL

## 3. JDBC 课堂七步法 → 本项目 dao 模板

课堂 JDBC 代码目录学习顺序：

```
1数据库连接 → 2添加 → 3修改 → 4查询 → 5不安全访问 → 6改造 → 7 PreparedStatement
```

本项目**只采用第 7 步及以后的安全写法**。

### 3.1 标准流程（与课堂一致）

```java
// 1. 加载驱动（MySQL 8+）
Class.forName("com.mysql.cj.jdbc.Driver");

// 2. 获取连接
Connection conn = DriverManager.getConnection(url, user, password);

// 3. 预编译 SQL（必须用 ? 占位符）
PreparedStatement ps = conn.prepareStatement("SELECT * FROM building WHERE id = ?");
ps.setString(1, id);

// 4. 执行
ResultSet rs = ps.executeQuery();  // 查询
// int rows = ps.executeUpdate();  // 增删改

// 5. 映射 ResultSet → model
while (rs.next()) {
    building.setId(rs.getString("id"));
    building.setLandArea(rs.getBigDecimal("land_area"));
}

// 6. 关闭资源（finally 或 try-with-resources）
```

### 3.2 课堂示例 vs 本项目

| 课堂写法 | 本项目规定 |
|----------|------------|
| `com.mysql.jdbc.Driver` | `com.mysql.cj.jdbc.Driver`（mysql-connector-j 8.x） |
| `Statement` + 字符串拼 SQL | **禁止**；用 `PreparedStatement` |
| `rs.getInt(1)` 按列号 | 推荐 `rs.getString("column_name")` 可读性更好 |
| `assignment` 库 / `tbStudent` 表 | `building_manos` 库 / `building` 等表 |
| 课堂用户 `java` / `java12345` | 本地自定，写入 `database.properties` |

### 3.3 参考文件

| 用途 | 路径 |
|------|------|
| 最简连接查询 | `course_example_java_learn/JdbcTest.java` |
| PreparedStatement 更新 | `JDBC代码/代码/7使用preparestatement/Test.java` |
| BaseDao 封装 | `鲜花商店/.../dao/BaseDao.java` |
| Dao 实现范例 | `鲜花商店/.../dao/impl/FlowerStoreDaoImpl.java` |

## 4. 控制台交互（课堂 Main.java 模式）

课堂鲜花商店 `Main.java` 使用的模式，本项目沿用：

```java
Scanner input = new Scanner(System.in);
while (running) {
    System.out.println("1. xxx  2. yyy  0. 退出");
    int choice = input.nextInt();
    switch (choice) {
        case 1 -> buildingService.add(...);
        case 0 -> running = false;
        default -> System.out.println("输入有误，请重新选择");
    }
}
```

本项目将上述逻辑集中在 `cli/MenuController` 和 `cli/ConsoleUtils`，与课堂相比：

- 输入校验封装到 `ConsoleUtils.readInt(min, max)`
- 列表展示用 `ConsoleUtils.printTable`
- **不在 cli 中写 SQL**

## 5. 接口与策略模式（课堂 → 折扣）

课堂鲜花商店：

```java
public interface Buyable { void buy(Flower flower); }
public interface Sellable { void sell(Flower flower); }
```

本项目购买折扣：

```java
public interface DiscountStrategy {
    BigDecimal apply(BigDecimal originalPrice);
    String getTypeName();
}
```

实现类：`PercentageDiscount`（课堂类比：固定比例）、`ThresholdDiscount`（满减）。

## 6. 异常处理（课堂要求）

课堂 `异常处理代码/` 要点，本项目应用：

| 课堂知识点 | 本项目用法 |
|------------|------------|
| `try-catch-finally` | dao 中关闭 Connection / PreparedStatement / ResultSet |
| `try-with-resources` | **推荐**替代 finally 手动 close |
| `throws SQLException` | dao 方法可声明抛出，service 捕获转业务提示 |
| `throw new IllegalArgumentException` | service 业务校验失败 |
| 自定义异常 | 可选 `BusinessException`，答辩可说明 |

**禁止**：`catch (Exception e) {}` 空捕获；至少 `e.printStackTrace()` 或日志，cli 层给用户友好提示。

## 7. 集合框架（课堂隐含使用）

鲜花商店 dao 广泛使用：

```java
List<FlowerStore> storeList = new ArrayList<>();
while (rs.next()) {
    storeList.add(flowerStore);
}
return storeList;
```

本项目所有查询方法返回 `List<Model>`，无结果时返回**空列表**（`Collections.emptyList()` 或 `new ArrayList<>()`），不返回 `null`。
