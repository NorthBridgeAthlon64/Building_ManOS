# 代码注释规范

> 采用 **标准 JavaDoc 标签**，IDE（IntelliJ IDEA / Eclipse / VS Code）可自动识别、生成文档与参数提示。  
> 课程要求：每个类注明**编写者**、**完成时间**、**具体功能**。

---

## 一、总体原则

| 原则 | 说明 |
|------|------|
| 标准标签优先 | 使用 `@author`、`@since`、`@param` 等，不写自定义 `【职责】` 块 |
| 简洁可读 | 类注释 2~4 句；方法首句说明做什么，复杂逻辑再补充 1 段 |
| 说清 IO 与算法 | 输入输出用 `@param` / `@return`；核心步骤写在描述正文里 |
| 行内注释克制 | 仅标注非显而易见的公式、业务规则、SQL 意图 |

---

## 二、常用 JavaDoc 标签

| 标签 | 用途 | 用于 |
|------|------|------|
| `@author` | 编写者姓名 | 类、接口 |
| `@since` | 完成日期或版本，如 `1.0` 或 `2026-07-15` | 类、方法（新增时） |
| `@version` | 当前类版本（可选） | 类 |
| `@param` | 参数含义、约束 | 方法 |
| `@return` | 返回值含义 | 方法 |
| `@throws` | 异常条件 | 方法 |
| `@see` | 关联类或方法 | 类、方法 |
| `{@link 类#方法}` | 可点击跳转链接 | 描述正文 |

IDE 会对以上标签做语法高亮、悬停提示和「生成 JavaDoc」补全。

---

## 三、类注释

每个 **public 类 / 接口 / 枚举** 必须有类级 JavaDoc。

### 模板

```java
/**
 * 楼盘业务服务，负责楼盘增删改查及删除前关联校验。
 * <p>
 * 由 {@link com.building.manos.cli.MenuController} 调用，
 * 通过 {@link com.building.manos.dao.BuildingDao} 访问数据库。
 * </p>
 *
 * @author 张三
 * @since 2026-07-15
 * @version 1.0
 * @see BuildingDao
 * @see Building
 */
public class BuildingService {
```

### 要求

- **第一段**：一句话说明职责（课程要求的「具体功能」）
- **第二段**（可选）：在分层中的位置、依赖谁（帮助队友理解调用链）
- **@author**：与 `docs/report/团队分工表.md` 一致
- **@since**：完成日期，格式 `yyyy-MM-dd`
- **@see**：关联的 dao、model、调用方（1~3 个即可）

## 四、方法注释

所有 **public / protected** 方法必填；含业务逻辑或算法的 **private** 方法建议填写。

### 简单方法

```java
/**
 * 根据编号查询楼盘。
 *
 * @param id 楼盘主键，非空
 * @return 存在则返回楼盘对象，否则 {@code null}
 */
public Building findById(String id) {
```

### 含业务步骤的方法

算法写在**描述正文**（`<p>` 内），标签只列 IO：

```java
/**
 * 删除指定楼盘。若楼盘下仍有房屋则拒绝删除。
 * <p>
 * 处理流程：先 {@link HouseDao#countByBuildingId(String)} 统计房屋数；
 * 为 0 时调用 {@link BuildingDao#deleteById(String)}。
 * </p>
 *
 * @param buildingId 楼盘编号
 * @return 删除成功 {@code true}，记录不存在 {@code false}
 * @throws IllegalStateException 楼盘下仍有房屋时
 */
public boolean deleteById(String buildingId) {
```

### 含计算公式的方法（如折扣）

```java
/**
 * 按比例计算折后价，公式：{@code 实付 = 原价 × rate}，{@code rate} 取值 (0, 1]。
 *
 * @param originalPrice 原价（元）
 * @return 折后实付金额（元）
 */
public double apply(double originalPrice) {
```

### 要求

| 项 | 说明 |
|----|------|
| 首句 | 方法做什么 |
| 正文 `<p>` | 仅在有校验顺序、SQL、公式、多步流程时补充 |
| `@param` | 每个参数：含义 + 是否可空 + 单位（如有） |
| `@return` | 返回值含义；说明 `null` / 空列表约定 |
| `@throws` | 列出业务异常及触发条件 |

## 五、实体类与字段

```java
/**
 * 楼盘实体，对应表 {@code building}。
 *
 * @author 张三
 * @since 2026-07-15
 */
public class Building {

    /** 楼盘主键 */
    private String id;

    /** 占地面积，单位：㎡ */
    private double landArea;
}
```

- 类级 JavaDoc 同普通类
- 字段用**单行** `/** */` 说明含义；单位、取值范围写在字段注释里

## 六、行内注释

仅在以下情况使用 `//`：

```java
// 总价 = 面积 × 单价
double totalPrice = area * unitPrice;

// status='ON_SALE'，价格在 [min, max] 闭区间内
PreparedStatement ps = conn.prepareStatement(sql);
```

避免：`// 定义变量`、`// 调用方法` 等无信息注释。

---

## 七、分层简要约定

| 包 | 类注释侧重 | 方法注释侧重 |
|----|-----------|-------------|
| `cli` | 菜单与终端交互，不直接访问数据库 | 用户输入项、调用的 service |
| `service` | 业务规则与流程编排 | 校验逻辑、调用的 dao、副作用 |
| `dao` | 对应哪张表，不含业务判断 | SQL 意图、参数与 ResultSet 映射 |
| `discount` | 策略用途 | 折扣公式（写在描述里） |
| `model` | 对应表名 | 一般无方法，字段单行注释 |

## 九、审查清单

**作者提交前：**

- [ ] 每个 public 类有 `@author`、`@since`、职责描述
- [ ] 每个 public 方法有 `@param`、`@return`（有异常则 `@throws`）
- [ ] 购买、折扣、删除等有分支逻辑的方法，描述里写了简要流程或公式

**合并前（组长）：**

- [ ] `@author` 与团队分工表一致
- [ ] 无大段无标签的裸注释代替 JavaDoc

---

## 十、与 IDE 配合

- **生成注释**：在方法上方输入 `/**` 后回车，IDE 自动补全 `@param` / `@return`
- **查看文档**：Ctrl + Q（IDEA）悬停查看 JavaDoc
- **跳转**：`{@link}` 与 `@see` 可 Ctrl + 点击跳转到目标类
