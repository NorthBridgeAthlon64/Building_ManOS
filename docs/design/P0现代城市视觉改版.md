# Building ManOS P0 现代城市视觉改版

## 1. 改版目标

本轮改版将界面从深色、香槟金主导的“豪华老钱感”，调整为更适合年轻城市资产团队的现代视觉：

- 使用暖白、钴蓝、湖青、珊瑚橙与紫色建立明快的城市色谱。
- 扩大页面、卡片、表格、表单与交互控件之间的间距，降低视觉拥挤。
- 通过轻量漂浮、脉冲、轨道动效与指针视差增强动态感。
- 在总览主标题旁加入 Three.js 城市住宅场景，让房屋元素成为核心信息载体。
- 保留现有 Mock 数据与前端业务流程，不连接真实后端。

## 2. 参考图转译原则

参考图中的城市建筑采用高密度体块、暖白立面和高饱和点色。本项目提取其视觉语言，但不直接复制具体建筑：

- 建筑主体：暖白和浅灰，降低传统豪宅材质感。
- 识别色：钴蓝窗带、湖青玻璃、珊瑚橙体块、紫色高层与芥末黄点缀。
- 造型语言：低多边形、清晰轮廓、阶梯露台、重复窗格和屋顶设备。
- 信息关系：模型作为主视觉，业务指标通过悬浮信息片和图例与模型建立联系。

## 3. 核心视觉规范

| 类别 | 方案 |
| --- | --- |
| 页面背景 | `#F3F6FA`，叠加极浅蓝青径向光晕 |
| 主色 | `#4F65F6` 钴蓝 |
| 动态辅助色 | `#18A8BA` 湖青 |
| 成交提示色 | `#ED6A46` 珊瑚橙 |
| 创意点缀色 | `#8A55D5` 紫色 |
| 面板 | 白色半透明、20–22 px 圆角、低对比描边 |
| 主视觉容器 | 32 px 圆角、蓝青渐变、柔和体积光 |
| 页面水平留白 | 桌面端 44 px，移动端 18 px |
| 常规模块间距 | 22–30 px |

## 4. Three.js 城市住宅场景

### 4.1 模型构成

- 中央主体为一栋带阶梯露台、玻璃窗带、屋顶设备和绿植的现代住宅。
- 周围布置不同高度的暖白、橙、紫红和芥末黄城市建筑，形成明确的前中后景。
- 建筑窗户使用实例化网格，减少重复绘制开销。
- 场景包含道路、公共平台、屋顶水箱、低多边形绿植及柔和阴影。

### 4.2 交互与动态

- 指针移动控制城市体块的轻微水平和垂直视差。
- 核心住宅具有低幅度漂浮动效，强化“实时资产模型”感受。
- 数据卡片、在线状态点和轨道图形使用节制的循环动画。
- 检测到 `prefers-reduced-motion` 时关闭持续运动，保留静态信息。

### 4.3 性能策略

- 总览页使用路由级懒加载，其他模块不会提前下载 Three.js。
- Three.js 独立拆分为可缓存分块。
- 像素比上限为 1.75，避免高分屏渲染成本无上限增长。
- 组件卸载时释放几何体、材质、渲染器、动画帧与监听器。

## 5. 模块视觉调整

| 模块 | 前端调整 |
| --- | --- |
| 全局导航 | 深色侧栏改为明亮半透明侧栏；选中项使用蓝色柔光胶囊 |
| 资产总览 | 新增 3D 城市住宅主视觉、悬浮信息片、彩色指标卡和项目脉冲列表 |
| 楼盘资产 | 卡片增大间距，建筑线稿由金色改为蓝紫城市体块语言 |
| 房源中心 | 筛选器、表格、房源卡片和详情抽屉统一为轻量圆角容器 |
| 成交工作台 | 步骤节点改为蓝色动态状态，业务区块间距扩大 |
| 成交账本 | 表格行高与单元格留白增加，金额信息使用主色聚焦 |
| 系统状态 | 深色主卡改为浅色科技面板，状态图标使用蓝、青、绿色 |

## 6. P0 边界

- 所有数据、增删改与成交操作仍由浏览器内 Mock Store 提供。
- Three.js 模型不映射真实地块坐标，也不代表具体建筑 BIM 数据。
- 本阶段不接入后端接口、地图服务、模型文件上传或实时协作。

## 7. V2 高饱和平涂方向

本轮进一步参考 Daniel Rich 建筑绘画中公开可见的高层特征：几何化城市结构、压缩空间、精确窗格和高饱和纯色。项目仅转译这些通用设计原则，不复刻具体作品。

- 页面骨架由钴蓝、湖青、珊瑚橙、紫色和黄色大色块构成，不再以白色面板作为唯一主体。
- 圆角和柔光阴影改为 2 px 硬描边、锐利转角和实体偏移阴影。
- 非总览页面标题区加入抽象城市立面；工作区空白处加入分层城市天际线。
- Three.js 城市扩展为 25 栋常规建筑和 1 栋中央住宅，并增加道路、斑马线、车辆、树木、街灯、阳台、立面层带、屋顶水箱和天线。
- 场景支持拖动旋转与全城/住宅视角切换。
