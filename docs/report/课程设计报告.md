## 摘要

本项目针对房地产公司房屋销售管理需求，设计并实现了一套基于 **Java** 与 **MySQL** 的控制台应用系统。系统采用 **cli → service → dao** 三层架构，通过 JDBC 完成数据持久化，实现了楼盘管理、房屋管理、多条件查询、折扣购买及销售记录等功能。项目由三人小组分工协作完成，满足课程大作业全部技术要求。

**关键词**：Java；MySQL；JDBC；分层架构；控制台应用；房屋销售管理

## 第 1 章 绪论

### 1.1 项目背景

房地产行业在日常运营中需要维护大量楼盘与房源信息，并记录成交数据。本课程大作业要求使用 Java 与数据库技术，在控制台环境下实现一套可演示、可答辩的管理系统。

### 1.2 项目目标

## 第 3 章 概要设计

> 详细内容见 [概要设计.md](../design/概要设计.md)

### 3.1 系统架构

系统分为表示层（cli）、业务层（service）、数据访问层（dao）及 MySQL 数据库四层。表示层通过菜单驱动用户交互；业务层封装业务规则；dao 层执行 SQL。

### 3.2 功能模块划分

| 模块 | 说明 |
|------|------|
| 楼盘管理 | 楼盘 CRUD |
| 房屋管理 | 房屋 CRUD、总价计算 |
| 房屋查询 | 多条件查询 |
| 房屋购买 | 折扣策略、成交处理 |
| 销售记录 | 历史成交查询 |

### 3.3 数据库设计概要

三表：`building`（楼盘）、`house`（房屋）、`sale_record`（成交记录）。楼盘与房屋为一对多，房屋与成交记录为一对一（简化模型）。

## 第 4 章 详细设计

> 详细内容见 [Java技术框架.md](../design/Java技术框架.md)、[数据库设计.md](../design/数据库设计.md)

### 4.1 类设计

- **model**：`Building`、`House`、`SaleRecord` 等实体  
- **dao**：`BuildingDao`、`HouseDao`、`SaleRecordDao` 及实现类  
- **service**：`BuildingService`、`HouseService`、`PurchaseService` 等  
- **cli**：`MenuController` 统一调度  
- **discount**：`DiscountStrategy` 策略模式实现折扣  

### 4.2 核心算法：购买与折扣

实付金额由折扣策略计算：

- **比例折扣**：`finalPrice = originalPrice × rate`  
- **满减**：满足条件时 `finalPrice = originalPrice - reduction`  

### 4.3 数据库表结构

见 [数据字典.md](../requirements/数据字典.md) 及 `sql/schema.sql`。
