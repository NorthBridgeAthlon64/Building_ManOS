## 摘要

本项目针对房地产公司房屋销售管理需求，设计并实现了基于 **Java** 与 **MySQL** 的房屋销售管理系统。系统采用 **cli/api → service → dao** 分层架构，通过 JDBC 完成数据持久化；表示层提供控制台与（可选）Vue + Javalin API。实现了楼盘管理、房屋管理、多条件查询、档位折扣购买、销售记录与看板等功能。文档终稿见《大作业报告》。

**关键词**：Java；MySQL；JDBC；分层架构；房屋销售管理

## 第 3 章 概要设计

> 详细内容见 [概要设计.md](../design/概要设计.md)

### 3.1 系统架构

系统分为表示层（cli）、业务层（service）、数据访问层（dao）及 MySQL 数据库四层。表示层通过菜单驱动用户交互；业务层封装业务规则；dao 层执行 SQL。

### 3.2 功能模块划分

| 模块 | 说明 |
|------|------|
| 楼盘管理 | 楼盘 CRUD |
| 房屋管理 | 房屋 CRUD、总价计算 |
| 房屋查询 | 多条件查询 |
| 房屋购买 | 折扣策略、成交处理 |
| 销售记录 | 历史成交查询 |

### 3.3 数据库设计概要

三表：`building`（楼盘）、`house`（房屋）、`sale_record`（成交记录）。楼盘与房屋为一对多，房屋与成交记录为一对一（简化模型）。

## 第 4 章 详细设计

> 详细内容见 [Java技术框架.md](../design/Java技术框架.md)、[数据库设计.md](../design/数据库设计.md)

### 4.1 类设计

- **model**：`Building`、`House`、`SaleRecord` 等实体  
- **dao**：`BuildingDao`、`HouseDao`、`SaleRecordDao` 及实现类  
- **service**：`BuildingService`、`HouseService`、`PurchaseService` 等  
- **cli**：`MenuController` 统一调度  
- **discount**：`DiscountStrategy` 策略模式实现折扣  

### 4.2 核心算法：购买与折扣

实付金额由 `DiscountStrategy` 按**房屋原价档位**计算（`PriceTier`）：

| 原价（元） | 比例折扣 | 满减 |
|------------|----------|------|
| &lt; 100 万 | ×1 | 减 2 万 |
| 100 万 ~ 300 万 | ×0.97 | 减 5 万 |
| ≥ 300 万 | ×0.92 | 减 15 万 |

购买在 `PurchaseService` 中开启 JDBC 事务，同步更新 `house` 为已售并插入 `sale_record`（`house_id` 唯一）。
