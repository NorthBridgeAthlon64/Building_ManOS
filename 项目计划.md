> 制定日期：2026-07-10

---

## 一、项目概述

实现某房地产公司的**房屋销售管理系统**：

- 楼盘管理（增删改查）
- 房屋管理（增删改查、多条件查询）
- 房屋购买（选房 → 折扣 → 成交）
- MySQL 数据持久化 + 四层分层架构（cli → service → dao → database）

> 详细需求见 [docs/requirements/需求分析.md](./docs/requirements/需求分析.md)

## 三、分层架构

```
┌─────────────────────────────────────┐
│  cli（表示层）                       │  MenuController, ConsoleUtils
├─────────────────────────────────────┤
│  service（业务逻辑层）               │  BuildingService, PurchaseService
├─────────────────────────────────────┤
│  dao（数据访问层）                   │  BuildingDao, HouseDao, SaleRecordDao
├─────────────────────────────────────┤
│  MySQL（数据库）                     │  building, house, sale_record
└─────────────────────────────────────┘

横切模块：model/（实体）、config/（DB连接）、discount/（折扣策略）、util/（工具）
```

**层间调用规则**：cli 只调 service；service 只调 dao；dao 不处理业务判断。

## 六、答辩要点

- 答辩时长 **≤ 5 分钟**，演示约 2 分钟
- 演示 3 步：查在售房源 → 购买九折成交 → 验证已售
- 详细大纲见 [ppt/答辩PPT大纲.md](./ppt/答辩PPT大纲.md)

### 常见答辩问题

| 问题 | 参考答案方向 |
|------|----------------|
| 分层架构如何体现？ | cli 只调 service，service 调 dao，dao 用 JDBC 访问 MySQL |
| 数据如何持久化？ | MySQL，`sql/schema.sql` 建表，dao 层 CRUD |
| 如何防止 SQL 注入？ | PreparedStatement 参数绑定 |
| 折扣如何实现？ | 策略模式 + service 层调用，结果写入 sale_record |
| 每个类的注释要求？ | JavaDoc 含 @author、@date、功能说明 |

## 七、原创性要求（避免零分）

两组代码完全一样将 **均为零分**，须主动差异化：

- 独立业务设计（字段、菜单、提示文案）
- 独立命名（包名、类名、方法名、表名）
- 独立折扣逻辑（至少 2 种策略）
- 独立 SQL 与 dao 实现
- 分工可追溯：每类 @author 不同，与分工表对应

## 八、风险与应对

| 风险 | 应对 |
|------|------|
| 答辩机无 MySQL | 提前确认环境；准备便携安装或录屏备份 |
| 未用数据库被扣分 | dao 层必须访问 MySQL，禁止仅用文件存储 |
| 注释/分工不符合要求 | 提交前用分工表逐类核对 @author |
| 与其他组代码雷同 | 合并前组长检查原创性 |
| 用了 GUI/Web | 严格控制台，代码评审时检查依赖 |
