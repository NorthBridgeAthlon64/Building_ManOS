## 一、5 分钟版设计原则

| 原则 | 说明 |
|------|------|
| **一页一事** | 能合并的必须合并，禁止 14 页流水账 |
| **演示优先** | 演示占 **2 分钟**，其余讲解 **≤ 3 分钟** |
| **图多字少** | 每页要点 ≤ 4 条，口述细节放演讲稿 |
| **得分点不漏** | Java、MySQL、分层、控制台、团队分工——各出现一次即可 |
| **快讲慢演** | 架构/数据库各 30 秒带过，演示步骤提前彩排 |

**推荐配色**：深蓝/白底；终端截图用等宽字体区域。

### 第 3 页｜技术设计（1 分钟）★合并页

**标题**：分层架构与数据库

**上半 — 架构图**（占页 55%）：

```
cli (MenuController)     ← ⏳ 开发中
    ↓
service (Building/House/Search/Purchase)  ← ✅ 邓单
    ↓
dao + config + model     ← ✅ 陈辉 · PreparedStatement
    ↓
MySQL (building / house / sale_record · uk_sale_house)
```

**下半 — 折扣档位 + 购买流程**：

| 原价 | 比例 | 满减 |
|------|------|------|
| &lt;100万 | 不打折 | 减2万 |
| 100~300万 | 97折 | 减5万 |
| ≥300万 | **92折** | 减15万 |

`选在售房 → 选折扣类型 → 档位计价 → 事务落库`

**口述要点**：
- 数据层与业务层已完成；菜单冲刺中
- 策略模式 + JDBC 事务；一套房一条成交记录
