# Building ManOS 答辩 PPT 大纲

> **读者**：PPT 组  
> **硬性限制**：**总时长 ≤ 5 分钟**（含演示）  
> **建议页数**：**6 页口述** + 1 页 Q&A 备注（可选，不口述）  
> **终稿存放**：`ppt/slides/Building_ManOS-答辩-v1.pptx`  
> **演讲稿**：见 [scripts/演讲稿.md](./scripts/演讲稿.md)

---

## 一、5 分钟版设计原则

| 原则 | 说明 |
|------|------|
| **一页一事** | 能合并的必须合并，禁止 14 页流水账 |
| **演示优先** | 演示占 **2 分钟**，其余讲解 **≤ 3 分钟** |
| **图多字少** | 每页要点 ≤ 4 条，口述细节放演讲稿 |
| **得分点不漏** | Java、MySQL、分层、控制台、团队分工——各出现一次即可 |
| **快讲慢演** | 架构/数据库各 30 秒带过，演示步骤提前彩排 |

**推荐配色**：深蓝/白底；终端截图用等宽字体区域。

## 二、逐页大纲（6 页口述版）

### 第 1 页｜封面（15 秒）

**标题**：Building ManOS — 房地产公司房屋销售管理系统

**副标题**：JAVA 高级编程大作业

**信息**：团队成员、答辩日期（教师名可选，不占口述时间）

---

### 第 2 页｜项目概述（40 秒）★合并页

**标题**：项目概述

**左栏 — 做什么**（3 条）：
- 控制台管理楼盘、房屋、成交记录
- 支持查询在售房源、折扣购买
- 数据持久化至 MySQL

**右栏 — 五大模块**（一行图标或五格）：

```
楼盘管理 │ 房屋管理 │ 房屋查询 │ 房屋购买 │ 销售记录
```

**底栏 — 课程要求对照**（小字表格，一扫而过）：

| Java 17 | MySQL+JDBC | cli→service→dao | 仅控制台 | JavaDoc+分工 |

**讲者提示**：30 秒讲业务，10 秒点课程要求，**不要逐条念表**。

---

### 第 3 页｜技术设计（1 分钟）★合并页

**标题**：分层架构与数据库

**上半 — 架构图**（占页 60%）：

```
cli (MenuController)
    ↓
service (BuildingService / PurchaseService)
    ↓
dao (BuildingDao / HouseDao)  ← PreparedStatement
    ↓
MySQL (building / house / sale_record)
```

**下半 — E-R 简图 + 购买 4 步**（一行流程）：

`选在售房 → 折扣计价 → 更新已售 → 写入 sale_record`

**口述要点**（各 15 秒）：
- 分层单向调用，cli 不写 SQL
- 三表关系：楼盘 1:N 房屋，成交写记录
- 折扣用策略模式 `DiscountStrategy`

---

### 第 4 页｜现场演示（2 分钟）★核心，不可压缩

**标题**：系统演示

**方式**：现场操作 **或** 预录屏（答辩机异常时播放）

**精简演示脚本**（只走 3 步，每步约 40 秒）：

| 步骤 | 操作 | 一句话说明 |
|------|------|------------|
| ① | 启动 → 查询在售房源 | 「数据来自 MySQL」 |
| ② | 购买一套房，九折成交 | 「service 算价，dao 落库」 |
| ③ | 再查该房 | 「状态已变为已售」 |

**PPT 素材**：主菜单截图 + 购买前后对比截图（左右分栏）；**不要**在 PPT 里列 6 步演示表。

**彩排要求**：技术组单独计时，**2 分钟内必须结束**。

### 与旧版（9 分钟）的差异

| 旧版 | 5 分钟版 |
|------|----------|
| 14 页 | 6 页口述 + 1 页 Q&A 可选 |
| 背景/功能/要求各一页 | 合并为「项目概述」 |
| 架构/数据库/流程/技术各一页 | 合并为「技术设计」 |

| 总结与 Q&A 分两页 | 合并，Q&A 放备注 |
